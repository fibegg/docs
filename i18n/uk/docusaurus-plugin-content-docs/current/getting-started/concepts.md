# Основні концепції

Розуміння ключових концепцій Playgrounds.dev.

## Playzone

**Playzone** — це ваш робочий простір верхнього рівня. Він підключається до GitHub-організації та керує всіма вашими середовищами. Вважайте його контейнером проекту.

## Playspec

**Playspec** визначає, як виглядає ваше середовище. Він базується на Docker Compose та описує ваші сервіси, бази даних, кеші та будь-яку іншу інфраструктуру, яка потрібна вашому додатку.

## Playground

**Playground** — це запущений екземпляр Playspec. Кожен playground отримує:

- **Ізольовані ресурси** — власні контейнери, мережі та томи
- **Унікальні субдомени** — кожен сервіс доступний за виділеним URL
- **Незалежний життєвий цикл** — запускайте, зупиняйте та видаляйте без впливу на інших

## Playroom

**Playroom** — це спільний простір, де можуть співіснувати декілька playground-ів. Він надає базову інфраструктуру (хост, мережа, DNS) для ваших playground-ів.

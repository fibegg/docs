# Чат з агентом

Чат з агентом дозволяє запустити **AI помічника для кодування загального призначення**, що працює на ваших особистих облікових даних, на будь-якій активній [Базі](/core-concepts/playroom) — без створення [Майданчика](/core-concepts/playground). Це найшвидший спосіб отримати AI-помічника на вашій інфраструктурі, використовуючи вашу власну підписку на провайдера.

## Початок чату

1. Перейдіть на сторінку деталей агента
2. Натисніть **Розпочати чат**
3. У модальному вікні оберіть активну **Базу** (лише не навчальні Бази)
4. За бажанням поставте позначку **Без терміну дії**, щоб вимкнути TTL
5. Натисніть **Запустити чат**

:::caution Один активний чат на агента
Лише **1 активний чат** дозволений на агента одночасно. Потрібно зупинити поточний чат перед початком нового.
:::

## Життєвий цикл чату

| Статус | Значення |
|---|---|
| **Очікує** (Pending) | Чат створено, очікує розгортання |
| **Розгортання** (Deploying) | Docker-контейнери завантажуються та запускаються на хості Бази |
| **Працює** (Running) | ✅ Чат активний — повноекранний iframe з інтерфейсом чату |
| **Помилка** (Error) | Розгортання не вдалося — повідомлення про помилку |
| **Зупинено** (Stopped) | Чат-сесію завершено (вручну або за закінченням TTL) |

### Що відбувається під час розгортання

1. Генерується Docker Compose файл спеціально для вашого чат-контейнера
2. Compose файл синхронізується на хост Бази через rsync
3. `docker compose pull` + `docker compose up -d` запускають контейнери
4. Оновлення статусу транслюються у ваш браузер в режимі реального часу через Turbo Streams

## URL чату

Кожен чат отримує унікальний субдомен:

```
chat-ac-{random}-{slug-агента}.{кореневий-домен-бази}
```

Чат захищений автоматично згенерованими обліковими даними HTTP Basic Auth.

## Термін дії та продовження

| Налаштування | Поведінка |
|---|---|
| **TTL за замовчуванням** | 2 години від початку чату |
| **Продовжити +2г** | Додає 2 години від поточного терміну закінчення (або від зараз, залежно що пізніше) |
| **∞ Безстроково** | Повністю видаляє TTL — чат працює необмежено (потрібне підтвердження) |

## Зупинка чату

Натисніть **Зупинити чат** на сторінці деталей агента або в заголовку чату. Це:

1. Зупиняє Docker-контейнери на віддаленому хості
2. Видаляє compose файл із Бази
3. Встановлює статус чату на `stopped`

## Автоматичне очищення

Фонове завдання запускається періодично для:

- **Зупинки прострочених чатів** — Активні чати, TTL яких закінчився
- **Очищення зависших розгортань** — Чати в стані `deploying` більше 10 хвилин позначаються як помилкові

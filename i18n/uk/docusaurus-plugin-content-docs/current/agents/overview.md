# Збережені агенти

Збережені агенти дозволяють **назавжди зареєструвати ваші особисті облікові дані AI провайдера** — з вашої власної підписки на Gemini, Claude Code чи OpenAI Codex — безпосередньо у Playgrounds.dev. Після збереження ви можете:

- **Підключати** їх до будь-якого [Майданчика](/core-concepts/playground), автоматично інжектуючи ваші облікові дані в sidecar-контейнер агента
- **Запускати AI чат загального призначення** на будь-якій з ваших [Баз](/core-concepts/playroom) одним кліком — без створення Майданчика
- **Повторно використовувати** їх у різних Майданчиках і чат-сесіях без повторної автентифікації

## Власні облікові дані (Bring Your Own Credentials)

На відміну від підходів на основі токенів, де платформа надає доступ до API, збережені агенти використовують **ваші особисті облікові дані підписки** від кожного провайдера. Це означає:

- **Ваша квота, ваші ліміти** — Усе використання AI працює на вашій власній підписці Gemini / Claude / OpenAI, тож ви отримуєте повну потужність свого тарифного плану
- **Жодних токенів платформи** — Playgrounds.dev ніколи не надає та не обмежує ваші AI токени; ви приносите свої
- **Повний доступ до функцій провайдера** — Оскільки ви автентифікуєтесь зі свого справжнього облікового запису, ви отримуєте доступ до всіх моделей, лімітів швидкості та функцій, що входять у ваш тарифний план
- **Постійне зберігання** — Облікові дані шифруються та зберігаються один раз; вони зберігаються між сесіями, Майданчиками та чат-сесіями, поки ви явно не відкличете їх

## Підтримувані провайдери

| Провайдер | Тип автентифікації | Файл облікових даних |
|---|---|---|
| **Gemini** | OAuth URL → вставити JSON | `oauth_creds.json` |
| **Claude Code** | Вставити OAuth токен | `agent_token.txt` |
| **OpenAI Codex** | Device code → вставити `auth.json` або API ключ | `auth.json` |

## Створення агента

1. Перейдіть до **Агенти** на бічній панелі
2. Натисніть **Новий агент**
3. Введіть **Назву** (напр. "Мій Gemini", "Робочий Claude")
4. Оберіть **Провайдера** (Gemini, Claude Code або OpenAI Codex)
5. Натисніть **Створити агента**

:::info Квота
Кожен обліковий запис може зберігати до **10 агентів** — наприклад, окремі агенти для особистого Gemini, робочого Claude та проєктного OpenAI Codex.
:::

## Статуси агента

| Статус | Значення |
|---|---|
| **Очікує** (Pending) | Створено, але ще не автентифіковано |
| **Автентифіковано** (Authenticated) | Облікові дані збережено, агент готовий до роботи |
| **Закінчився** (Expired) | Термін дії облікових даних закінчився |
| **Відкликано** (Revoked) | Облікові дані відкликано вручну |

Агент **може використовуватися**, якщо його статус `authenticated` і необов'язковий термін дії ще не настав.
